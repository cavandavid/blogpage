Beyond Basics

Errors detected during execution are called as Exceptions. All StandardError's originate from class Exception which in turn inherits from class BaseException. There are plenty of errors in Python, in this module we will go through some of the more common ones.
1. ValueError

This error occurs when a function receives an argument that is of correct type but is of incompatible value to a function.
For eg.
print (int("99")) >>99 print (int("im here to break your code :)")) >>ValueError: invalid literal for int() with base 10: 'im here to break your code :)

NOTE: int function accepts string objects, it is the value of the string object that causes the opertaion to fail


2. TypeError

This error occurs when a function receives an object of incorrect type i.e incompatible to the function.
For eg.
print (int([1,2,3,4,5])) >> TypeError: int() argument must be a string, a bytes-like object or a number, not 'list'

3. IndentationError

This error occurs when we have inconsistency in the indentation of our code.
For eg.
def do_something(): try: print ("Hello") except: do_something() >> do_something() ^ IndentationError: expected an indented block
Here have left our except block empty. It is not allowed to have empty blocks, use pass instead which literally does nothing and is just used as a filler. The above code snippet can be rewritten as
def do_something(): try: print ("Hello") except: pass do_something() >> Hello

4. NameError

This error occurs when we call a function or a variable before it has been defined. for eg.
print (displaymessage()) def displaymessage(): return 'Welcome, good sir' >>NameError: name 'displaymessage' is not defined

Here we have called displaymessage before defining it

5. SyntaxError

This error is commonly seen amongst new Python developers , so if you see it don’t freak out. It occurs when our code is not syntactically correct
while True print('Check your syntax') >>while True ^ SyntaxError: invalid syntax
NOTE: The missing colon in front of the while condition

6. IndexError

This error occurs when we try to access an indexed element from an iterable series like strings, lists etc which doesn’t exist or is out of bounds.
my_list=[1,2,3,4,5] print (my_list[10]) >>print (my_list[10]) IndexError: list index out of range
Here We are trying to access the 10th element from a list that has only 5 elements.

7. KeyError

This error occurs when we the compiler is unable to find the specified key in the dictionary.
my_dict={'Jon':'Targaryen','Sansa':'Stark','Arya':'Stark'} print (my_dict['Ramsey']) >>print (my_dict['Ramsey']) KeyError: 'Ramsey'
Here We are trying to access the key 'Ramsey' which is not present in the dictionary.

8. ImportError

This error occurs when we the compiler is unable to find the module we are trying to import in our program.This can be due to a spelling error or the module has not been installed on your system.
import teme >>import teme ModuleNotFoundError: No module named 'teme'
Perhaps a drunk man trying to type the module time

9. ZeroDivisionError

This error occurs when we try to divide by 0.
def our_divider(num): return 100/num print (our_divider(0)) >> return 100/num ZeroDivisionError: division by zero
It is better to catch this as if we succeed in our division by zero we will end up tearing a hole in time and space.

10. NotImplementedError

Remember abstract methods from our days in Java? Well in old school Python before the days of abc module, definining abstract methods was a bit lengthy.
class animal(): def talk(self): print ('Hello good sir, im an animal') class dog(animal): #class dog is inheriting from class animal def talk(self): print ('Wuff wuff!') d=dog() d.talk() >> Wuff wuff!
In the above example, we have initialized class dog, however nothing is stopping us from initializing parent class animal and calling its talk method, which breaks the fundamental concept of an abstract class. Since as we know abstract classes cannot be initialized and abstract methods should generally get their definition from the child class. In this case our NotImplementedError comes in handy, rewriting above code as.
class animal(): def talk(self): raise NotImplementedError("You cannot initialise abstract class") class dog(animal): def talk(self): print ("Wuff wuff!") d=animal() d.talk() >>raise NotImplementedError("You cannot initialise abstract class") NotImplementedError: You cannot initialise abstract class


NOTE: Not every exception is exceptional enough to be caught, infact it is the responsibility of the programmer to decide which exception should be caught and which shouldn't.
In general, you should avoid catching IndentationError, SyntaxError , NameError as they are due to programmer mistakes.
